# このリポジトリでの作業開始時ルール

このリポジトリで新しいスレッドを開始したら、まず [docs/codex-notes.md](docs/codex-notes.md) を読むこと。

以後の作業では、`docs/codex-notes.md` に書かれた共通指示・注意点・学習メモを前提に振る舞うこと。

## 作業開始時

1. 最初に `docs/codex-notes.md` を確認する。
2. 既存シミュレーターの文脈、表記、UI方針、実装上の注意をそこで確認する。
3. 指示が競合しない限り、その内容を優先して合わせる。

## 作業中

- 既存ページのデザイン言語、操作感、文言トーンをできるだけ揃える。
- 新しく学んだ注意点や、同じミスを防ぐための知見が出たら、作業の区切りで `docs/codex-notes.md` に追記する。
- 追記は、既存の方針を壊さないよう簡潔にまとめる。

## ユーザーが `以上。` と言ったとき

ユーザーが `以上。` のように、作業が一段落した合図を出したら、そのスレッドで学んだことを振り返ること。

その際は、必要に応じて `docs/codex-notes.md` の

- `今回学んだこと`
- `実装上の注意`
- `UI/UXの注意`

などの節に追記して、次のスレッドで再利用できる形にすること。

## 注意

- 追記は、今回の作業から一般化できる内容だけにする。
- 一時的な事情や、その場限りのメモは入れない。
- 既存の未コミット差分には触れない。
- 文字コードを壊しやすいファイルでは、編集方法にも注意する。
