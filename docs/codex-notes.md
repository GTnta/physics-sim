# Codex ノート

このファイルは、このリポジトリで Codex が作業するときに学んだこと、再発防止したいこと、次回すぐ思い出したいことを追記していくためのメモです。

原則:
- 新しい学びは下に追記する
- 長文より、次に役立つ短いルールで書く
- 事実と推測を分ける
- 具体的なファイル名や状況を書く
- 他のシミュレーターでも使える内容を優先して残す
- 個別事例を書くときは、再利用できる一般則と分けて書く
- 追記する前に、追記予定の内容をユーザーへ提示する
- ユーザーが内容を確認してから追記する
- 追記は承認を得てから行う

## 基本方針

- 既存の公開済みシミュレーターと同じサイト内の一機能として作る
- `index.html` からたどれる公開物として整える
- UI、説明文、ボタン、ラベル、注記は日本語を基本にする
- 物理として不自然な挙動や誤学習につながる見せ方は避ける
- 初学者が迷わず触れることを優先する

## 実装ルール

- 手作業の編集は `apply_patch` を使う
- 検索は `rg` を優先する
- 作業対象外の未コミット変更には触れない
- HTML を変更したら、可能なら表示確認まで行う
- 作り上げるために必要な構文確認、表示確認、ローカルブラウザ確認、簡易的な動作確認は、追加承認なしで行ってよい
- ローカル HTML の表示確認としてヘッドレス Edge を実行する場合も、追加承認なしで行ってよい
- 無駄な `requestAnimationFrame` の常時実行は避ける

## UI/UX ルール

- ヒーローにある情報を左パネルに重複して置かない
- 操作対象は見た目で明確に分かるようにする
- モード切替、表示切替、補助操作は見た目の階層を分ける
- 凡例や説明文が主表示領域を圧迫しないようにする
- iPad 横置きで、主表示と主要操作が一画面に収まることを優先する
- 16:9 の画面サイズを意識し、主キャンバスと主要操作パネル部分だけを拡大表示したとき、それらが一画面に収まる構成を優先する
